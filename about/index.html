关于博客

为什么选择自己做博客，而不是选择博客站点

从 2010 年接触博客一晃已经过去了好多年。

刚开始用的其实跟大多数人一样是个博客站点，当时热情很高，曾经做过的笔记一一发布到博客上。 还把珍藏的口琴谱拿出来分享，因为有些口琴谱是自己根据曲子还原出来的谱子，在网上找不到的，给博客带来了不错的访问量，有时候一天能有个上千的访问量。

2012 年走完青藏线，把路上的点点滴滴通过博文的方式分享了出去。下半年赶上工作忙有段时间没有写作，等到年底的时候傻眼了。有些文章被删除了。 可能是内容的原因，也可能是文中的照片不合规，总之不翼而飞了。

当时还从来没想过自己做博客，愤怒之后的第一个想法是博客搬家，于是折腾半天把博客搬到了新浪。 每种博客都有自己的特点，不可能马上能适应，原来习以为常的功能在新博客系统里找不到，再加上之前博客站点对博文任意删除等行为的影响，虽然新浪博客站点也很不错， 但是没能再次激起我使用的欲望。因为如果你发的内容在一个博客站点上是不合规的，那么在另一个博客站点也必定是不合规的，毕竟是在一个体系内。

于是萌生了搬到国外博客站点的念头。不过很快就失望了。整体功能上要弱了很多，甚至有些博客站点的功能连国内博客站点的一半都达不到。

最后，痛定思痛，参考了一些牛人的做法之后，最后选择了 github 静态站。博客内容可以按自己喜欢的方式去写，功能强不强大主要靠自己开发，对于程序员来说还是蛮不错的选择。 于是最后，我选择了自己做博客，而不是用现成的博客站点。

博客都发些什么内容

作为程序员肯定少不了技术文章，算是自己成长进步的一个脚印吧。回头看看之前写过的文章还是蛮有意思的。

有一阵挺喜欢订阅和阅读公众号上的文章，看到精彩之处总是要收藏回头再看一看。 也许是因为公众号的文章亦是博文，也免不了同样的命运，曾经收藏的文章有时候就再也无缘看到了。 于是开始将不错的、有感触的留在博客上，至少给那些博文一些肯定它们曾经很受欢迎。

平时除了偶尔吹吹口琴，也喜欢弹弹钢琴，自学虽然进步很慢但过程还是挺享受的。 除了口琴谱之外，这些年收藏了挺多不错的钢琴谱，分享出去是不是会有更多人享受其中呢。
